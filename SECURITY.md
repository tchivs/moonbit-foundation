# Security Policy

MoonBit Native Foundation currently ships pre-1.0 candidate source. Security
reports are accepted for the documented public packages and repository
automation; no response-time, embargo-duration, bounty, or production-support
commitment is made.

## Report a vulnerability

Prefer the repository's private vulnerability-reporting route:

- [Open a private vulnerability report](https://github.com/moonbit-foundation/moonbit-foundation/security/advisories/new)

If that private form is unavailable, open a minimal
[public routing issue](https://github.com/moonbit-foundation/moonbit-foundation/issues/new)
that contains only the affected module, a statement that security impact may
exist, and a request for a private contact route. Do not include exploit steps,
proof-of-concept code, credentials, private data, or unpatched technical detail
in the public issue.

Include exact versions and targets, impact, prerequisites, and a minimal
reproduction only in the private report. The sole maintainer will validate and
prioritize the report when available, may request more evidence, and will record
any fix and disclosure decision in repository history. This process grants no
registry authority and never requires a reporter to share credentials.

Unsupported or deferred features, dependency-only vulnerabilities without an
MNF impact path, and claims based solely on an unobserved Mooncakes rendering
may be redirected or closed.
