# Security Policy

MoonBit Native Foundation currently ships pre-1.0 candidate source for
`tchivs/mb-core`, `tchivs/mb-color`, and `tchivs/mb-image`. Security reports
cover the documented public packages and local repository automation; no
response-time, embargo-duration, bounty, or production-support commitment is
made.

## Report a vulnerability

Preserve a security report locally and privately until a verified reporting
channel exists. Include exact versions and targets, impact, prerequisites, and
a minimal reproduction. Do not put exploit steps, proof-of-concept code,
credentials, private data, or unpatched technical detail into a public channel.

The intended repository metadata is
`https://github.com/tchivs/moonbit-foundation`. The repository, private advisory
form, and public issue route are **unverified and not operational** until a
read-only existence proof succeeds. No email or alternate external route is
currently asserted. Once a channel is verified, the sole maintainer can validate
and prioritize reports and record fixes and disclosure decisions in repository
history. This process grants no registry authority and never requires a reporter
to share credentials.

Unsupported or deferred features, dependency-only vulnerabilities without an
MNF impact path, and claims based solely on an unobserved Mooncakes rendering
may be redirected or closed.
