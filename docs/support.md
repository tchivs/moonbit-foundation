# Support

MoonBit Native Foundation modules are pre-1.0 candidates. Support covers the
documented public packages, pinned toolchain floor, and required targets listed
in each module README. Candidate APIs may change under the compatibility policy;
there is no stability, response-time, or production-readiness commitment.

## Request support

Use the repository's public issue route:

- [Open a support issue](https://github.com/moonbit-foundation/moonbit-foundation/issues/new)
- Include the module and package name, exact version, `moon`, `moonc`, and
  `moonrun` versions, target, a minimal reproduction, expected result, and
  actual result.
- Remove credentials, tokens, private source, personal data, and other secrets
  before posting.

The sole maintainer triages reports when available. A report may be closed or
redirected when it concerns deferred scope, unsupported targets, unpublished
registry state, or a security vulnerability. Security-sensitive reports must
follow [the security-reporting route](../SECURITY.md), not a public issue.

Registry publication and Mooncakes-rendering claims are outside this support
route until publication is performed and independently observed.
