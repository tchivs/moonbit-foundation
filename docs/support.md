# Support

MoonBit Native Foundation modules `tchivs/mb-core`, `tchivs/mb-color`, and
`tchivs/mb-image` are pre-1.0 candidates. Support covers the documented public
packages, pinned toolchain floor, and required targets listed in each module
README. Candidate APIs may change under the compatibility policy; there is no
stability, response-time, or production-readiness commitment.

## Request support

Until an external read-only existence check succeeds, record a support request
locally with the checkout and keep it available for later triage. Include the
module and package name, exact version, `moon`, `moonc`, and `moonrun` versions,
target, a minimal reproduction, expected result, and actual result. Remove
credentials, tokens, private source, personal data, and other secrets.

The intended repository metadata is
`https://github.com/tchivs/moonbit-foundation`. That repository and its issue
route are **unverified and not operational**. Do not submit a report to the
intended issue URL until a read-only repository and route existence proof has
passed.

The sole maintainer triages reports when available. A report may be closed or
redirected when it concerns deferred scope, unsupported targets, unpublished
registry state, or a security vulnerability. Security-sensitive reports must
follow [the security-reporting route](../SECURITY.md), not a public issue.

Repository support, registry publication, and Mooncakes-rendering claims remain
outside the verified support surface until each is independently observed.
