the three independently versioned MoonBit Native Foundation modules. It is a
compatibility and evidence index, not a publication notice or a stable API
declaration.

## Public package DAG

The module dependency order is `mb-core -> mb-color -> mb-image`. Public package
edges are closed and acyclic:

- mb-core: `error`; `checked -> error`; `budget -> checked + error`;
  `bytes -> budget + checked + error`; `io -> budget + bytes + checked + error`;
  `host -> checked + error`.
- mb-color: `model -> mb-core/error`; `transfer -> model`;
  `quantize -> model + mb-core/error + checked`;
  `alpha -> model + quantize + mb-core/error + checked`;
  `profile -> mb-core/error + budget + bytes`.
- mb-image: `metadata -> mb-core/error + budget + bytes`;
  `model -> metadata + mb-core/error + checked + budget + mb-color/model + profile`;
  `storage -> model + metadata + mb-core/error + checked + budget + bytes + mb-color/model + profile`;
  `ops -> storage + model + metadata + mb-core/error + checked + budget + bytes + mb-color/model + alpha + profile`;
  `codec -> storage + model + metadata + mb-core/error + budget + bytes + io`;
  `ppm -> codec + storage + model + metadata + mb-core/error + checked + budget + bytes + io + mb-color/model + profile`.

There is no umbrella module, root facade, path dependency, reverse dependency,
or global codec registry.

## Reference codec boundary

The concrete codec is named the **MNF strict PPM P6/sRGB subset**. It accepts one
binary P6 image with positive decimal width and height, `maxval` exactly `255`,
and exactly `width * height * 3` encoded-sRGB RGB bytes, followed by EOF. Its
canonical encoder writes `P6\n<width> <height>\n255\n` plus tight logical RGB
rows. This scope does not establish full Netpbm colorimetry, multi-image PPM,
arbitrary maxval scaling, P3, 16-bit PPM, or production codec breadth.

## Deferred and prohibited claims

The candidate does not claim a stable API, registry publication, full PPM
conformance or colorimetry, production PNG/JPEG/WebP breadth, performance
superiority, or required LLVM support. LLVM remains an experimental,
non-blocking lane outside the required target set.

Also deferred are wider PPM variants, animation, native/system codecs, a global
registry, ambient filesystem/URL policy, signed releases, GPU/tiled storage,
advanced resampling, gamut mapping, full ICC parsing/transforms, rendering, and
the v2 graphics/document modules.

Public publication remains blocked while
`policy/foundation.json.publication.blocked` is true and owner namespace
verification is false. Publishing is a later explicit action and is not part of
this candidate documentation gate.
