## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller-owned PNG bytes to `PngDecoder.decode` | Untrusted binary framing, length fields, chunk types, payloads, CRCs, and metadata policy enter a forward-only parser. |
| PNG structural gate to future raster output | Phase 20 validates every future resource envelope but must not expose an image until Phase 21 owns DEFLATE and scanlines. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-20-01 | Denial of service | Chunk length and IHDR arithmetic | high | mitigate | Reject declared lengths above `0x7fffffff` before payload handling, then use fixed scratch/discard reads, per-byte input ceiling, checked arithmetic, codec ceilings, and non-consuming Budget child validation in `structural.mbt`. |
| T-20-02 | Tampering | Chunk type, order, and integrity | high | mitigate | Validate four ASCII type letters and uppercase reserved third byte before classification; then enforce private state, CRC-32 over type plus payload, first/only IHDR, contiguous IDAT, empty IEND, and one-byte strict EOF. |
| T-20-03 | Tampering | Semantic and opaque metadata | medium | mitigate | Explicit known-semantic rejection; unknown ancillary data is CRC-checked and preservation requests return a typed capability error. |
| T-20-04 | Elevation of privilege | Parser-to-image handoff | medium | mitigate | `decode` returns only the typed pending-capability error after validation and constructs no `OwnedImage`; Phase 21 is the sole owner of decompression/raster success. |
| T-20-05 | Supply chain | MoonBit package dependencies | low | accept | No external package installation; use existing workspace modules only. |
