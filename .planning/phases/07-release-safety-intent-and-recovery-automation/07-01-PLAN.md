<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Git/ref and tracked source -> release intent | Movable, dirty, or mismatched source could be authorized under a trusted-looking name. |
| Qualification evidence -> canonical digest | Ambiguous serialization or a hash cycle could produce unstable or selectively bound authority. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-07-01 | Spoofing/Tampering | ref/source binding | high | mitigate | Peel the dedicated tag when checked, compare exact source SHA, reject dirty/untrusted source, and bind both values in schema. |
| T-07-02 | Tampering | canonical intent | high | mitigate | Closed schema, controlled canonical profile, independent digest recomputation, and two-clean-copy equality. |
| T-07-03 | Information Disclosure | intent generator | high | mitigate | Credential-free inputs only plus static and dynamic rejection of credential paths, token fields, and raw output. |
| T-07-04 | Repudiation | digest semantics | medium | mitigate | Policy and README state SHA-256 is content identity; GitHub actor/ref authorization remains a separate Phase 7 control. |
