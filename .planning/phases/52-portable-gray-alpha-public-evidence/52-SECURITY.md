# Phase 52 — Security

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Public image/encoder input to PNG bytes | Non-symmetric GrayAlpha source components cross the public `PngEncoder` profile and serialization boundary into a standards-visible type-4 raster. | Gray and alpha bytes, PNG framing, and compatibility vectors. |
| Encoder state to caller-owned mutable lease | The caller selects lease capacity and owns every unwritten byte while public `PngChunkEncoder::pull` reports accepted output and terminal state. | Output prefixes, lease tails, cumulative progress, and terminal outcome. |
| Public PNG decoder to decoded image | Type-4 sample pairs cross the public decoder canonicalization boundary. | GrayAlpha8 samples to straight RGBA8 pixels. |

The public `PngDecoder` then proves straight RGBA8 gray replication and unchanged alpha for both pairs. Evidence: `modules/mb-image/png/encode_test.mbt:152-176`, `383-413`, `919-944`; all six public eager pairs also decode through this helper at `952-977`. | closed |
