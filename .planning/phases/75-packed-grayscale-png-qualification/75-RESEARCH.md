## Summary

Phase 73 already proves that the public eager selectors produce the expected IHDR fields and literal packed Stored scanlines for odd widths: Gray1 `00 55 80`, Gray2 `00 1b 00`, and Gray4 `00 0f 10`. Its parser is independent of production packing and decoding helpers, but every PNG under that assertion originates from the encoder being tested; it does not independently prove that an externally authored packed Type-0 PNG decodes to the canonical RGBA8 view. [VERIFIED: 73-VERIFICATION.md; modules/mb-image/png/encode_test.mbt]

Phase 74 already provides the necessary all-depth hostile transport evidence: zero/small/ragged output capacities, accepted-byte accounting, untouched lease tails, eager/chunk byte identity, atomic invalid-level and work-budget construction, and sticky released-lease terminals. Existing eager frozen vectors also cover Gray8, Gray16, GrayAlpha8/16, generic RGB8, and generic RGBA8 routes. Phase 75 should retain those tests rather than recreate lifecycle machinery or change production files. [VERIFIED: 74-VERIFICATION.md; modules/mb-image/png/stream_encode_test.mbt; modules/mb-image/png/encode_test.mbt]

**Primary recommendation:** Add one qualification-only test slice in `png_test.mbt` containing three complete, hand-authored, CRC-valid non-interlaced Type-0 PNG literals (depths 1/2/4) and static canonical RGBA8 expectations. Use those same static vectors, not encoder-generated bytes, for eager full-wire equality; retain the existing Phase 74 lifecycle and legacy-vector tests unchanged, then run `moon -C modules/mb-image test png --target all --frozen`. [VERIFIED: 75-CONTEXT.md; modules/mb-image/png/png_test.mbt; modules/mb-image/png/encode_test.mbt; modules/mb-image/png/stream_encode_test.mbt]

## Architecture Patterns

### System Architecture Diagram

~~~text
hand-authored complete Type-0 PNG literal (1 / 2 / 4 bit)
  -> public generic PngDecoder
  -> canonical RGBA8 pixels + descriptor/metadata assertions

the same static literal
  <- public PngEncoder::new_gray{1,2,4} from matching canonical Gray/U8 source
  <- PngChunkEncoder::new_gray{1,2,4} through retained hostile leases

legacy RGB/RGBA/Gray8/Gray16 frozen bytes
  -> retained existing compatibility assertions

all above
  -> moon -C modules/mb-image test png --target all --frozen
  -> wasm | wasm-gc | js | native
~~~

### Pattern 1: Single static vector, two public boundaries

**What:** Define one complete literal per depth, including PNG signature, IHDR, Stored-DEFLATE IDAT, valid CRCs, and IEND. Decode the literal through `PngDecoder::new()` and compare every output RGBA8 channel to a separately spelled expected sequence. Then compare the matching public eager selector's entire output to that literal. [VERIFIED: 75-CONTEXT.md; modules/mb-image/png/png_test.mbt; modules/mb-image/png/encode_test.mbt]

**When to use:** Use only where the output contract is fixed to Stored/None/non-interlaced bytes, as the Gray1/2/4 selectors are. [VERIFIED: modules/mb-image/png/png.mbt; 73-VERIFICATION.md]

**Vector matrix:**

| Depth | Required odd-width fixture | Filtered bytes that the independent literal contains | Canonical decoded gray levels |
|---:|---|---|---|
| 1 | width 9: `00 ff 00 ff 00 ff 00 ff ff` | `00 55 80` | `00 ff 00 ff 00 ff 00 ff ff` |
| 2 | width 5: `00 55 aa ff 00` | `00 1b 00` | `00 55 aa ff 00` |
| 4 | width 3: `00 ff 11` | `00 0f 10` | `00 ff 11` |

[VERIFIED: modules/mb-image/png/encode_test.mbt; modules/mb-image/png/raster_decode.mbt]

### Pattern 2: Retain shared hostile lifecycle proof

Do not reimplement a drain or lease mock. Keep `png_stream_packed_hostile_drain` and its `[0,1]`, `[1]`, and ragged-capacity schedules, plus its atomic-admission and released-lease sticky-terminal tests. These already invoke every public low-bit chunk factory and make byte identity, ownership, totals, terminal replay, and unchanged budgets observable. ## Common Pitfalls

### Pitfall 1: Encoder-produced bytes used as the decoder oracle

**What goes wrong:** An encode-then-decode round trip can pass when encoder and decoder share the same bit-position mistake. [VERIFIED: 75-CONTEXT.md]

**How to avoid:** Use complete hand-authored literals, literal packed bytes, and independently spelled canonical output samples. [VERIFIED: 75-CONTEXT.md; modules/mb-image/png/png_test.mbt]

### Pitfall 2: Testing only completed chunks

**What goes wrong:** Full-capacity output overlooks zero-capacity non-progress, unaccepted-tail mutation, acknowledgement accounting, and terminal replay. [VERIFIED: modules/mb-image/png/stream_encode_test.mbt]

**How to avoid:** Retain the existing zero/one/ragged schedules, sentinel tails, and released-lease replay checks for every depth. [VERIFIED: modules/mb-image/png/stream_encode_test.mbt]

### Pitfall 3: Replacing legacy vectors while adding low-bit coverage

**What goes wrong:** A broad test rewrite can silently drop fixed RGB/RGBA or Gray8/16 byte baselines. [VERIFIED: 75-CONTEXT.md; modules/mb-image/png/encode_test.mbt]

**How to avoid:** Add the qualification fixture beside the current suite and leave frozen legacy vector assertions intact. [VERIFIED: 75-CONTEXT.md]
