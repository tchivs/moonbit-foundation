# Phase 69 Plan 01: Explicit RGBA16 PNG Encoding

## Objective

Add the narrow eager-only, non-interlaced Type-6/16 RGBA16 encoder profile by extending the existing profile-aware bounded machine and prove exact U16 byte order plus atomic rejection.

## Artifacts this phase produces

- `PngEncodeProfile::Rgba16` private profile case.
- `PngEncoder::new_rgba16`, `new_rgba16_with_compression_strategy`, `new_rgba16_with_filter_strategy`, and `new_rgba16_with_strategies` public eager factories.
- RGBA16 profile-aware Type-6/16 IHDR, preflight, and filtered-raster byte handling.

<action>
First add failing public tests using a checked non-symmetric `ImageFormat::rgba16()` source. Assert `PngEncoder::new_rgba16()` emits IHDR bit depth `0x10`, colour type `0x06`, interlace `0x00`, and an independently stated normal-row raster order `Rhi,Rlo,Ghi,Glo,Bhi,Blo,Ahi,Alo`; assert `PngDecoder::decode_rgba16` restores the original component-byte order. Then add `Rgba16` to `PngEncodeProfile`, add the four eager factory constructors in `png.mbt`, route them through `PngEncodeMachine::new_with_profile` with `PngInterlaceStrategy::None`, select colour type 6 and eight-byte-per-pixel accounting in `stream_encode.mbt`, and make the profile-aware raster cursor emit high then low bytes for every rgba16 component in `encode.mbt`.
</action>
<acceptance_criteria>

- The new test observes Type-6/16 IHDR and the exact eight PNG wire bytes per source pixel.
- `moon -C modules/mb-image test png --target js --frozen --filter '*rgba16*'` passes.
</acceptance_criteria>
