## Resolved research decision

The Phase 13 research question about QOI decode work is resolved for implementation: `decode_work = checked_add(pixel_count, output_bytes)`. Both operands are validated from the 14-byte header before allocation. That one checked value is the `CodecLimits.max_work` request and the `OwnedImage::new_operation` work charge, so the existing `Budget` preflight either charges the complete operation atomically or leaves all counters untouched. It deliberately excludes compressed-byte and per-opcode work, which cannot be fully derived before forward-only decoding starts; reader error progress remains an I/O concern.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller prefix → `QoiDecoder.probe` | Untrusted caller-owned bytes are classified without I/O or allocation. |
| forward-only `Reader` → decoder | Untrusted compressed header/chunk/marker bytes and progress outcomes drive parsing. |
| parsed header → allocation/budget | Untrusted dimensions and channel declarations can request resource use. |
| generated fixture source → MoonBit test table | Repository JSON is transformed into checked test code. |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-13-01 | Denial of Service | QOI header preflight | high | mitigate | Checked width/height/pixel/output/work arithmetic; all `CodecLimits` and descriptor checks precede `OwnedImage::new_operation` and caller budget charge. |
| T-13-02 | Denial of Service | QOI chunk reads | high | mitigate | Enforce actual consumed-input ceiling before each read and use `@io.read_exact` to reject EOF, zero progress, and invalid progress deterministically. |
