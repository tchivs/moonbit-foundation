# Phase 23 Plan 02: Fixed Colour Length Safety Summary

Fixed-size sRGB, gAMA, and cHRM PNG declarations now reject invalid header lengths before any payload read or attacker-controlled buffering, with compact all-target regressions for 2 GiB declared chunks.

## Completed Work

- Task 1 (`179698f`): added a fixed-colour length selector and pre-payload singleton/length gates in `_png_read_colour_chunk`; valid 1-, 4-, and 32-byte values retain CRC-authenticated semantic parsing, while iCCP remains streamed and unretained.
- Task 2 (`f201b25`): added manifest-backed `2147483647` header-only sRGB, gAMA, and cHRM cases. The generator independently validates that form, emits bytes ending immediately after the colour type, and regenerates the portable decoder corpus.

The generated hostile vectors assert `png-srgb-length`, `png-gama-length`, and `png-chrm-length` respectively. Since each record has no declared payload or CRC bytes, a regression to payload-first parsing would produce an end-of-stream error instead.
