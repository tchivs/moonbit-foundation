# v0.11 Milestone Audit — PNG Dynamic Huffman Compression

**Status:** passed  
**Audited:** 2026-07-22

## Definition of Done

MoonBit PNG callers can opt into Dynamic compression without changing Stored or FixedOrStored compatibility output. The encoder selects a bounded Dynamic DEFLATE plan only for a strict complete-PNG size win, replays it safely through eager and caller-buffered APIs, and proves the behavior with generated RGB8/RGBA8 cases on js, wasm, wasm-gc, and native.

## Integration and End-to-End Flow

The independent integration review found five critical handoffs wired and no missing connections:

1. `DynamicOrFixedOrStored` flows from both public factories through `PngEncodeMachine` to shared preflight.
2. The canonical Huffman builder feeds the bounded Dynamic plan.
3. Preflight retains FixedOrStored and selects Dynamic only for a strict complete-PNG win.
4. The selected plan reaches both eager and caller-buffered adapters through acknowledgement-safe replay.
5. Generated RGB8/RGBA8 bytes decode through the public `PngDecoder` with descriptor and component fidelity.

Four end-to-end flows are complete: eager dynamic selection, hostile-capacity chunk output, acknowledgement-gated replay through PNG framing, and generated-corpus round trips. Independent evidence recorded 22/22 focused Dynamic tests and 131/131 full PNG tests on each of wasm, wasm-gc, js, and native.
