### Generic Compatibility

The original generic paths remain separate: `PngDecoder::decode` and
`PngChunkDecoder::new` select the generic profile, while the new APIs are
explicit opt-ins. Phase 68 tests the same independent Type-6/16 inputs through
both paths and retains `RGBA8(Rhi,Ghi,Bhi,Ahi)` projection. **WIRED.**

### Profile, Resource, and Lifecycle Safety

`Rgba16` admission requires Type 6/16, opaque transparency, and only default
or sRGB declarations; legacy colour facts and ICC are rejected before output
budget, descriptor, image, sink, lifecycle, or result creation. Preflight uses
eight output bytes per pixel. Chunk `push` reports accepted bytes only; a
successful `finish()` is the only result transfer, while failed and finished
states reject later input with zero consumption. Exact/one-less normal and
Adam7 resource tests exercise this boundary. **WIRED.**
