# Validate decode of non-interlaced low-bit-depth grayscale PNG

## Objective

Extend the existing bounded grayscale PNG decode path to color type 0 at bit
depths 1, 2, and 4, retaining the depth-8 behavior. Packed source scanlines
must be reconstructed in their encoded byte representation and then expanded to
RGB8 or straight RGBA8 when a valid grayscale `tRNS` key is present.

The work is the grayscale/transparent portion of PNGX-01 only. It excludes
palette work, grayscale-alpha, 16-bit samples, Adam7, color-management,
encoding changes, public streaming APIs, QOI, release, and configuration work.

## Scope coverage audit

| Source | Item | Coverage |
|---|---|---|
| GOAL | Decode low-bit non-interlaced grayscale with exact packed filtering and RGB/RGBA expansion | Tasks 1-3 |
| REQ | PNGX-01 grayscale and transparency profile with explicit image-model mapping | Tasks 1-3 |
| CONTEXT | Depth 1/2/4 plus retained depth 8; packed row byte formula; bpp-1 filters; private checked row buffers; MSB-first actual-width unpack; tRNS high-byte/mask checks; exhaustive evidence | Tasks 1-3 |
| RESEARCH | Existing pure-MoonBit decoder, resource-budget, generated-fixture, and four-target patterns are sufficient (Level 0) | Existing package patterns applied in Tasks 1-3 |

## Atomic file ownership

This remains one Quick plan: IHDR/preflight facts, row-cache ownership,
streaming and white-box reconstruction, generated source cases, and manifest
evidence are one behavioral contract. Dividing them would allow stale generated
records or a decoder whose output-budget calculation disagrees with its packed
scanline path.

## Threat model

| Boundary | Description |
|---|---|
| untrusted IHDR/tRNS to PNG transport | Bit depth, packed row size, and key validity determine allocation and alpha semantics. |
| compressed IDAT to packed-row raster state | Attacker-controlled bytes and filter tags determine packed predictor state before image writes. |
| JSON corpus/generator to generated tests | Fixture declarations, oracle output, and manifest digest become conformance evidence. |

| Threat ID | Category | Component | Severity | Disposition | Mitigation plan |
|---|---|---|---|---|---|
| T-260721-5j4-01 | Denial of service | grayscale preflight and packed-row allocation | high | mitigate | Checked row/image/work arithmetic and one atomic Budget child reserve output plus exactly two packed rows before allocation; test exact and below limits. |
| T-260721-5j4-02 | Tampering | packed filter reconstruction | high | mitigate | Reconstruct with bpp 1 in private current/previous byte rows, validate filter tags and emitted byte count, and test every filter across byte boundaries. |
| T-260721-5j4-03 | Information integrity | low-bit expansion and tRNS comparison | high | mitigate | MSB-first actual-width unpack, depth-specific scaling, raw-key comparison, and high-byte/mask validation are proved by accepted and hostile vectors. |
| T-260721-5j4-04 | Tampering | generated PNG evidence | medium | mitigate | Independent PowerShell oracle, exhaustive split generation, manifest-digest check, and four-target PNG tests make fixture drift observable. |

## Success criteria

- Color type 0 at depths 1, 2, 4, and 8 accepts only non-interlaced legal inputs and yields exact RGB8 or tRNS-derived straight RGBA8 output.
- Packed source row sizing, bpp-1 filtering, two private budgeted rows, MSB-first actual-width unpacking, scaling, and raw tRNS comparison are tested at public and white-box boundaries.
- All filters, byte-crossing widths, exhaustive two-IDAT splits, malformed tRNS keys, fixture integrity, and all four portable targets are automatically verified.
- No QOI, release, registry, configuration, encoding, palette, Adam7, color-management, 16-bit, grayscale-alpha, or public streaming work is included.
