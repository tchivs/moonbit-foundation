# Validate decode of non-interlaced 8-bit indexed PNG

## Objective

Extend the portable eager PNG decoder with the bounded indexed-color subset:
non-interlaced, 8-bit color type 3 with exactly one valid PLTE before IDAT and
without tRNS. Reconstructed indexes map to RGB8. This is a further subset of
PNGX-01; palette-alpha, low bit depths, 16-bit, Adam7, and color-management
support remain outside the task.

## Tasks

<task type="auto" tdd="true">
  <name>Task 1: Parse PLTE and reserve indexed transport before image output</name>
  <files>modules/mb-image/png/structural.mbt, modules/mb-image/png/png.mbt, modules/mb-image/png/structural_wbtest.mbt</files>
  <action>Accept type 3 only with bit depth 8 and interlace 0. For that type only, accept exactly one CRC-valid PLTE before the first IDAT, store its exact bytes and entry count in `PngStreamTransport`, and make that transport the sole palette source passed to `_png_inflate_zlib_to_raster`; preserve PLTE rejection for type 0, 2, and 6. Enforce the exact predicate `length >= 3 && length <= 768 && length % 3 == 0`: lengths 3 and 768 are valid while 0, 1, 2, 4, 5, and 769 are generated `png-plte-length` failures. Reject missing PLTE at first IDAT with `png-plte-required`, repeated or post-IDAT PLTE with `png-plte-order`, and bad PLTE CRC with `png-crc`. Keep `tRNS` and colour-management chunks rejected as semantic chunks. Before calling `OwnedImage::new_operation`, create one checked Budget child/reservation for filtered input, RGB8 output, and exactly two width-byte index rows; pass that same child to the image allocation and both budget-bound index-row allocations in a fixed order, so a shortage in any one leaves all three unallocated and no result exposed. Account bytes, allocations, allocation size, and work. Keep type 0/2/6 arithmetic and behavior byte-for-byte unchanged.</action>
  <verify><automated>moon -C modules/mb-image test png --target native --frozen</automated></verify>
  <done>Type-3 transport cannot enter IDAT without an in-range palette or without all RGB output and private index-row resources being preflighted and charged before image allocation.</done>
</task>

<task type="auto" tdd="true">
  <name>Task 2: Filter reconstructed indexes then map palette entries to RGB8</name>
  <files>modules/mb-image/png/deflate_inflate.mbt, modules/mb-image/png/raster_decode.mbt, modules/mb-image/png/raster_decode_wbtest.mbt, modules/mb-image/png/png_test.mbt</files>
  <action>For indexed transport, receive only `PngStreamTransport.palette` and `palette_entries`, retain separate current and previous reconstructed index rows allocated from the transport's preflight reservation, and apply filter tags 0..4 with bpp 1 before palette lookup. Map each index to its PLTE triple only after proving it is less than the actual entry count; an index equal to entry count must fail `png-palette-index` without exposing a completed image. Never derive a filter neighbour from RGB output. Preserve direct RGB/RGBA and grayscale branches. Add public and white-box tests covering all filters with nonzero left/above/upper-left, index 0, last valid palette entry, one-past-last index, type-3 `tRNS`, type0/2/6 regression bytes, and exact/below indexed budget ceilings for bytes, allocations, allocation-size, and work with no result on failure.</action>
  <verify><automated>moon -C modules/mb-image test png --target all --frozen</automated></verify>
  <done>All indexed filter predictors use index lanes, every accepted pixel is a verified palette lookup, and the RGB8 result/limits remain observable only after atomic preflight.</done>
</task>
