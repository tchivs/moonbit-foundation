# Quick Task 260721-7d5 Summary

Implemented strict bounded decoding of non-interlaced 16-bit PNG colour types 0 and 2 into existing RGB8/RGBA8 images by emitting each big-endian sample's high byte and retaining full raw tRNS equality.

## Delivered

- Type-0 and type-2 IHDR validation accepts depth 16 while types 4 and 6 remain depth-8-only.
- Private transports carry encoded source bytes per pixel; type 0 uses bpp 2 and type 2 uses bpp 6 for both eager and live IDAT filter reconstruction.
- The only 16-bit image mapping is `out8 = raw16 high byte`; there is no scaling, rounding, dithering, or colour-management transform.
- Raw two-byte grayscale and six-byte RGB tRNS keys are preserved and compared before high-byte emission, so equal high bytes with different low bytes remain opaque.
- Added a PowerShell-independent oracle, corpus schema v2.4.0, manifest digest, and 863 generated unsplit/two-IDAT-split vectors covering successes, hostile transport input, and output/work limits across all portable targets.
