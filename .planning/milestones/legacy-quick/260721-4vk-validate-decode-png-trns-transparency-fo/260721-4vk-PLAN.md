# Validate decode of PNG tRNS transparency for supported 8-bit profiles

## Objective

Extend the currently supported non-interlaced, 8-bit PNG subset with standard `tRNS` handling: grayscale key transparency for colour type 0, RGB key transparency for type 2, and palette-alpha transparency for type 3. Accepted transparent images expose straight RGBA8; opaque type 0, 2, 3, and type 6 behavior remains unchanged.

This is a partial PNGX-01 slice. It does not complete the requirement's 16-bit profile, grayscale-alpha, low-bit-depth palette, Adam7, colour-management, public streaming, QOI, release, or configuration work.

| RESEARCH | Existing PNG decoder, indexed-row budget, fixture, and four-target patterns are sufficient | Tasks 1-3 |

## Atomic file ownership

All files are intentionally one Quick-plan change: the parser's transparency state, descriptor/preflight reservation, stream and white-box raster writers, public tests, source fixtures, independent generator oracle, generated tables, and manifest digests must agree. Splitting them could produce stale generated evidence or permit a decoder contract whose resource accounting no longer matches its fixture claims.

## Threat model

| Boundary | Description |
|---|---|
| untrusted PNG chunks to transport state | tRNS ordering, CRC, payload size, and high-byte rules decide whether alpha semantics are valid. |
| filtered IDAT bytes to RGBA output | Filter reconstruction and transparency comparison must not conflate source channels with output channels. |
| fixture JSON/generator to generated evidence | Declared expected pixels and checked-in vectors become conformance inputs. |

| Threat ID | Category | Component | Severity | Disposition | Mitigation plan |
|---|---|---|---|---|---|
| T-260721-4vk-01 | Tampering | `_png_read_stream_transport` | high | mitigate | CRC every tRNS payload before validation, require one legal pre-IDAT occurrence, and test malformed/order/duplicate/type matrices. |
| T-260721-4vk-02 | Denial of service | RGBA8 preflight and indexed cache reservation | high | mitigate | Checked arithmetic and one child Budget reserve four-channel image storage plus both index rows before allocation; test exact and below boundaries. |
| T-260721-4vk-03 | Information integrity | filter and transparency raster writers | high | mitigate | Reconstruct source bpp first, compare reconstructed keys or palette entries afterward, and test all filters across arbitrary IDAT boundaries. |
| T-260721-4vk-04 | Tampering | PNG fixture generation | medium | mitigate | Require independent oracle comparison, generated-artifact checks, both manifest digests, and four-target PNG tests. |
