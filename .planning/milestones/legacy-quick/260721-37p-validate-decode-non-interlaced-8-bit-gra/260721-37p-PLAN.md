# Validate decode of non-interlaced 8-bit grayscale PNG

## Objective

Extend the strict eager PNG decoder to accept the deliberately bounded colour-type-0 subset: 8-bit, non-interlaced, opaque grayscale PNG expands to RGB8 (`g,g,g`) without changing RGB/RGBA output, untagged colour semantics, chunk-policy rejections, or resource safety.

This is the scoped first portion of future requirement PNGX-01. It does not add palette, grayscale-alpha, 16-bit, Adam7, tRNS, or colour-management support, and it must not modify or revert the unrelated in-progress QOI files.

## Threat model

| Boundary | Description |
|---|---|
| untrusted PNG bytes → IHDR preflight | Dimensions, colour type, and source/output byte counts determine allocation and work charges. |
| compressed IDAT → scanline writer | Attacker-controlled DEFLATE output and filter bytes cross into image storage. |
| fixture JSON/generator → checked-in test vectors | Generated bytes and manifest digests become repository verification inputs. |

| Threat ID | Category | Component | Severity | Disposition | Mitigation plan |
|---|---|---|---|---|---|
| T-260721-37p-01 | Denial of service | grayscale IHDR preflight | high | mitigate | Use checked products and charge RGB8 allocation/output plus grayscale filtered work before allocation; test exact and below ceilings. |
| T-260721-37p-02 | Tampering | IDAT filter reconstruction | high | mitigate | Validate filter tags, preserve exact expected emitted count and Adler/CRC completion, and test all five filters across IDAT boundaries. |
| T-260721-37p-03 | Information integrity | colour semantics | medium | mitigate | Preserve encoded-sRGB/builtin-sRGB for untagged input and retain typed rejection of tRNS/gAMA/cHRM/iCCP/sRGB rather than silently interpreting them. |
| T-260721-37p-04 | Tampering | generated PNG evidence | medium | mitigate | Require generator `-Check`, manifest digest update, and four-target PNG tests; generated files are never hand edited. |
