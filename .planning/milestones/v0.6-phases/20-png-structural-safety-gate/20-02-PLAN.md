  <files>modules/mb-image/png/png.mbt, modules/mb-image/png/structural.mbt, modules/mb-image/png/png_test.mbt, modules/mb-image/png/structural_wbtest.mbt</files>
  <behavior>
    - A decoder reads signature, each four-byte big-endian length, four-byte type, payload, and CRC with fixed private scratch storage while incrementing the input counter per reader byte; a declared length above `0x7fffffff` rejects before payload access.
    - A chunk type reaches classification only when all four bytes are ASCII letters and its third byte is uppercase ASCII. CRC-32 starts with its type bytes, includes every payload byte, and must equal the big-endian stored CRC for mandatory, rejected-semantic, and discarded-ancillary chunks.
    - The state admits exactly one first 13-byte IHDR, then one or more contiguous IDAT chunks, then one empty IEND with valid CRC and immediate EOF. It rejects later IHDR, missing or separated IDAT, IEND before IDAT, nonempty/incomplete IEND, duplicated terminal data, and any post-IEND byte.
    - IHDR accepts only positive 8-bit type-2/type-6 dimensions with compression, filter, and interlace fields zero; it uses checked arithmetic for pixels, pixel bytes, filtered rows, output, work, and allocation envelope, verifies all CodecLimits, and calls an uncharged Budget child preflight.
    - Unknown critical and known palette, transparency, colour-management, HDR, and animation chunks return deterministic typed errors after structural validation; unknown ancillary chunks are CRC-checked then either discarded when opaque preservation is disabled or return the typed opaque-preservation capability error when it is enabled.
    - Only a fully structurally accepted transport returns `CapabilityUnavailable` with operation `png-decode` and context `deflate-and-raster-pending`; every error and that terminal result leave caller Budget and Diagnostics unchanged and expose no image.
  </behavior>
  <action>Replace the signature-only `_png_validate_transport` seam with the private `PngChunkState` machine described above. Per D-03, retain the existing one-byte owned scratch allocation and add fixed-size field readers that compose big-endian length/CRC values without allocating from untrusted lengths; a small fixed IHDR field accumulator is permitted, but neither a chunk-sized buffer nor an IDAT buffer may be created. Make `_png_read_one` the sole reader path so checked `consumed + 1` and `max_input_bytes` apply to signature, headers, data, CRC, and the final EOF probe. After each length field, perform the `0x7fffffff` ceiling test; after the type field, enforce four ASCII letters and an uppercase reserved third byte before reading a payload or choosing its policy. Update CRC incrementally with the type and each payload byte, read the stored CRC as big-endian, and compare before applying chunk-state transitions or returning a semantic/metadata rejection.

Per D-01 and D-03, parse IHDR only in `BeforeIhdr`, require length 13 and positive width/height, accept depth 8, colour type 2 or 6, and three method values zero. Derive pixel count, channel-adjusted bytes, filtered-row transport bytes, prospective output bytes, work, and allocation size through `@checked`; compare each applicable value to CodecLimits and use `Budget::child(ResourceLimits::new(...))` solely as a complete inherited preflight. Do not charge that child, allocate an `OwnedImage`, or push Diagnostics. Per D-04, encode the known semantic chunk set explicitly, route unknown critical chunks to data failure after CRC, and route unknown ancillary chunks through `preserve_opaque_metadata` only after CRC and bounded discard. Per D-02, leave `PngDecoder` as the sole public type and keep `decode`'s sole accepted terminal as `@codec.capability_unavailable("png-decode", "deflate-and-raster-pending")` after strict IEND-to-EOF validation.

Write public and white-box tests before implementation using the QOI MemoryReader/scripted-reader patterns.
