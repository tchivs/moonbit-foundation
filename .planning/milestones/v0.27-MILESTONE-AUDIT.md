## Phase Handoff

Phase 83's summary frontmatter declares the selected Type-3/1, /2, and /4 Adam7 eager/chunk selectors plus profile-aware preflight and replay as its provided contract. The live code confirms the eager facade in `encode.mbt` and the chunk facade in `stream_encode.mbt` both use the same private profile mapper and `PngEncodeMachine::new_with_indexed_profile`.

Phase 84's only changed implementation artifact is `stream_encode_test.mbt`. Its tests construct `PngChunkEncoder::new_indexed_with_interlace_strategy(..., Adam7, ...)`; they do not recreate an encoder or route through a legacy non-interlaced constructor. This is the required Phase 83 → Phase 84 connection.

The shared preflight/replay geometry is also real, not merely imported: preflight calls `_png_adam7_passes(width, height, 1UL, depth)` before limits and the sole budget charge, while replay calls the same geometry helper and fetches canonical source indices through `PngIndexedImage::index_at`. This closes the planning-to-emission data path without staging or a second encoder.
